# RiskAuth 2FA

Веб-приложение на Python для регистрации пользователя, входа по паролю, подтверждения через TOTP-код и оценки риска попытки входа.

Проект не требует установки внешних библиотек. Используются только стандартные модули Python.

---

## 1. Запуск

```bash
python app.py
```

После запуска открыть в браузере:

```text
http://127.0.0.1:5000
```

Для остановки сервера нажать `Ctrl + C` в терминале.

---

## 2. Что есть в проекте

- регистрация пользователя;
- хранение пароля в виде PBKDF2-хэша с солью;
- генерация секрета для TOTP;
- проверка 6-значного TOTP-кода;
- оценка риска входа по нескольким признакам;
- блокировка входа при высоком риске;
- сохранение попыток входа в SQLite;
- журнал входов в веб-интерфейсе;
- экспорт журнала входов в CSV;
- отдельная страница с демо-сценариями.

## 5. Балльная модель риска

| Признак | Баллы |
|---|---:|
| Новый IP-адрес | +20 |
| Новый браузер или устройство | +15 |
| Вход ночью | +10 |
| 1–2 неудачные попытки входа | +5 за каждую |
| 3+ неудачные попытки за последние 15 минут | +25 |
| Демо-подозрительная внешняя сеть | +30 |

---

## 6. Уровни риска

| Баллы | Уровень | Действие |
|---:|---|---|
| 0–29 | низкий | переход к 2FA |
| 30–59 | средний | переход к 2FA, запись в журнал |
| 60+ | высокий | вход запрещается |

---

## 7. Как проверить обычный вход

1. Запустить приложение командой `python app.py`.
2. Открыть `http://127.0.0.1:5000`.
3. Перейти на страницу регистрации.
4. Создать пользователя.
5. Скопировать TOTP-секрет в Google Authenticator, Microsoft Authenticator или Aegis.
6. Перейти на страницу входа.
7. Ввести логин и пароль.
8. Ввести 6-значный код из приложения-аутентификатора.
9. После успешного входа откроется кабинет пользователя.

---

## 8. Как проверить высокий риск

Высокий риск получается при сумме 60 баллов и выше.

Пример:

| Фактор | Баллы |
|---|---:|
| Новый браузер | +15 |
| Новый IP-адрес | +20 |
| Демо-подозрительная сеть | +30 |
| Итого | 65 |

Для проверки можно использовать адрес:

```text
http://127.0.0.1:5000/login?demo_ip=8.8.8.8
```

При высоком риске приложение покажет сообщение о запрете входа, а попытка появится в журнале.

## 9. Где смотреть основные функции в коде

| Функция | Назначение |
|---|---|
| `password_hash()` | создание хэша пароля |
| `check_password()` | проверка пароля |
| `new_totp_secret()` | создание секрета для 2FA |
| `totp()` | генерация одноразового кода |
| `verify_totp()` | проверка 2FA-кода |
| `calculate_risk()` | расчёт баллов риска |
| `risk_level_by_score()` | определение уровня риска |
| `decision_by_level()` | выбор действия по уровню риска |
| `log_attempt()` | запись попытки входа в журнал |
| `remember_device()` | сохранение доверенного устройства |
| `user_stats()` | статистика для кабинета пользователя |

## 10. Примечание

Это учебный прототип. Для реального использования нужно дополнительно добавить HTTPS, CSRF-защиту, полноценное управление сессиями, ограничение частоты запросов и хранение секретного ключа только в переменных окружения.
